[Coffee] New York Times: Windows Vista

2006-12-21

在 OIKOS 看到 Vista沒有抄OS X喔！？這篇文章，內容是 New York Times 一篇對 Vista 的評論 Windows Vista - David Pogue’s Review，這位記者特別把 Vista 與 Mac OSX 中相似的功能做了整理，以諷刺的語氣說明為什麼 Vista “並沒有” 抄襲 Mac OSX。回過頭來發現在 PTT Mac 版看到也有人轉貼，於是針對內容提了些個人的感想。

這位記者主要提到 Vista 的三個部份是抄襲 Mac OSX 的，包括了 Search / Spotlight、Sidebar & Gadgets / Dashboard & Widgets、還有西洋棋這個遊戲。

針對 Search / Spotlight 這個功能，我覺得這個概念的發想應該不能說是抄襲自 Mac，而是有搜尋以來就希望有的功能，但是 OSX 的確是第一個挑明了這個需求並在電腦系統中滿足它的系統。在這個概念提出後，Google 馬上推出了 Google Desktop，而 Linux 上也有類似的 Beagle 和 Kat 計畫開始進行。也就是說，目前市面上的主流系統都已經內建了這個功能，而且它的確是大家所需要的，Vista 現在才加進這個功能，要說抄襲實在太過沈重，我寧可說 Windows 終於比較跟上時代了。

Sidebar & Gadgets / Dashboard & Widgets，這位記者可能覺得功能相同，因此諷刺 Vista 甚至連 Gadgets 這個名稱都要抄襲 Widgets。但是我必須說，這種隨時展示一些資訊的小程式其實一直都有，例如 Windows 上的許多程式都會在系統列中顯示一些資訊，也有許多可以偽裝成桌面的小程式，例如時鐘、行事曆等等，Linux 上也有類似的計畫。但是這些軟體都有兩個嚴重的問題，第一個就是寫作難度，這些軟體都要求使用者幾乎是要寫個完整的視窗程式。這問題在 Mac 上的 Konfabulator 推出後獲得解決（雖然它後來被 Apple 內建 Dashboard 作掉了），利用網頁技術加上 Javascript，甚至搭配 AJAX，的確大大降低了撰寫難度。第二個問題是這類軟體在如何顯示上其實有相當的難度。舉 Windows 上的桌面行事曆來說，行事曆一定會被桌面上的視窗擋住，所以要看行事曆，就必須按下工作列上的顯示桌面，然後看著 Windows 很努力的把最上層的視窗縮小，把下一層的視窗顯示出來、重繪視窗內容、重繪完成之後又要把視窗縮小，一直重複直到最後把桌面顯示出來，看完之後使用者又要很努力的自己把所有的視窗開啟。整個動作非常的花時間，假如是要臨時很快的瞄一下行事曆，實在是非常的難過。再不然就要像 Google Desktop 一樣的方式，佔據螢幕上一個固定的區塊，但是這樣對螢幕的使用又非常的沒效率。這個問題 Mac 的 Dashboard 就解決了，它是另外 “蓋” 一層畫面在螢幕上，因此呼之即來，揮之即去，非常快速方便。現在回過頭來看 Vista 的 Sidebar…它採用的是類似 Google Desktop 那種固定佔用螢幕空間的方式，但是卻把這條 bar 放得很寬。也難怪會有人說 Vista 是要讓使用者多花些錢買寬螢幕，然後拿來當成窄螢幕使用。因此要說 Vista 的 Sidebar & Gadgets 抄襲 Dashboard & Widgets，我想是太抬舉 Vista 了，兩者就使用上的方便性而言實在是完全不能比。

但是第三項的 3D 西洋棋，我就真的要說，Vista 實在是抄的太蠢了。先別說 Windows 中從來沒有內建過西洋棋的遊戲，現在整個遊戲的界面，甚至連棋盤能換的紋路都幾乎和 OSX 中的一樣，實在是蠢到家。我覺得 Vista 把 Windows 傳統上內建的接龍改成能換紋路都比抄個西洋棋好。

總的來說，Vista 是個標準的微軟產品，非常符合微軟長久以來的企業文化，也就是東抄西抄，但卻抄的只得其形未得其神，徒具形象卻得不到精髓。
